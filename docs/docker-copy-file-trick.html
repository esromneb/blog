docker-copy-file-trick

Dec 10, 2023

Docker and Files

Docker is great for setting up build environments. However if you are trying to make a docker with a pre-installed IDE or some other software, a problem emerges. How do you use the COPY or ADD commands without leaving behind a layer.

Layers to the rescue

FROM ubuntu:jammy-20230308 as builder

COPY CCS12.5.0.00007_linux-x64.tar.gz /opt/CCS12.5.0.00007_linux-x64.tar.gz

RUN cd /opt && tar -xf CCS12.5.0.00007_linux-x64.tar.gz

###############################################################################
FROM ubuntu:jammy-20230308 as final

COPY --from=builder /opt/CCS12.5.0.00007_linux-x64 /opt/CCS12.5.0.00007_linux-x64

# And the rest of your docker file as normal

Results

A temporary docker, called builder is created. The COPY command is used here (which must leave behind a layer). Then the contents are untarred. In this step you can do any additional setup that is required only for aquiring the install files. Then in final, you only copy over the extracted folder.
